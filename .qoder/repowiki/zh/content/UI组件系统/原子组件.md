## 简介
原子组件是观己应用UI系统中最基础的构建单元，遵循原子设计方法论。这些组件是不可再分的最小UI元素，用于构建更复杂的分子组件和有机体组件。本文档详细描述了六个核心原子组件：胶囊状文本输入框、自增长文本编辑器、圆形图标按钮、可选标签芯片、粗滑块和标签输入芯片，涵盖其设计特性、功能实现和在不同功能模块中的复用模式。

## 胶囊文本编辑器 (CapsuleTextEditor)

胶囊文本编辑器是一种多行文本输入组件，采用胶囊状设计，提供简洁的输入体验。该组件支持iOS和macOS平台，并在iOS上通过`UIViewRepresentable`包装`UITextView`以实现更精细的控制。

**核心特性**:
- **圆角设计**: 组件采用圆角矩形设计，通过`.clipShape(Capsule())`实现平滑的边缘。
- **占位符行为**: 虽然代码中未直接实现占位符，但通过外部绑定的`text`属性和`TextEditor`的默认行为，当文本为空时显示占位内容。
- **输入辅助工具栏**: 在iOS上，组件集成了自定义的`inputAccessoryView`，包含添加照片、文件和收起键盘的按钮。
- **平台适配**: 使用`#if canImport(UIKit)`条件编译，为iOS和macOS提供不同的实现。

**关键属性**:
- `@Binding public var text: String`: 双向绑定的文本内容，实现数据同步。
- `public var onAddPhoto: () -> Void`: 添加照片的回调函数。
- `public var onAddFile: () -> Void`: 添加文件的回调函数。
- `public var onCollapse: () -> Void`: 收起键盘的回调函数。

**使用示例**:
```swift
// 文件路径: UI/Atoms/CapsuleTextEditor.swift
CapsuleTextEditor(
    text: $text,
    onAddPhoto: { /* 添加照片 */ },
    onAddFile: { /* 添加文件 */ },
    onCollapse: { /* 收起键盘 */ }
)
```

## 自增长文本编辑器 (GrowingTextEditor)

自增长文本编辑器是一种高度动态调整的文本输入组件，能够根据内容自动扩展其高度，提供流畅的输入体验。

**核心特性**:
- **高度动态调整**: 通过`@Binding public var dynamicHeight: CGFloat`属性，组件能够将内部`UITextView`的内容高度传递给父组件，实现容器高度的动态调整。
- **自动尺寸计算**: 在iOS上，通过继承`UITextView`并重写`intrinsicContentSize`属性，确保视图的固有内容大小能够正确反映文本内容的高度。
- **布局更新**: 在`Coordinator`的`textViewDidChange`方法中，每当文本发生变化时，都会异步更新`dynamicHeight`绑定值，触发UI重新布局。

**关键属性**:
- `@Binding public var text: String`: 双向绑定的文本内容。
- `@Binding public var dynamicHeight: CGFloat`: 双向绑定的高度值，用于父组件调整布局。

**实现机制**:
1. 当用户输入文本时，`UITextView`的内容发生变化。
2. `Coordinator`的`textViewDidChange`方法被调用，获取新的内容高度。
3. 通过`DispatchQueue.main.async`异步更新`dynamicHeight`，避免在布局过程中直接修改状态。
4. 父组件接收到高度变化，重新计算布局，实现平滑的高度调整。

## 圆形图标按钮 (RoundIconButton)

圆形图标按钮是一种简洁的图标按钮组件，提供点击反馈和良好的可访问性支持。

**核心特性**:
- **点击反馈**: 通过包装在`Button`中，利用系统默认的点击反馈（如高亮、动画），提供直观的交互响应。
- **可访问性支持**: 组件继承了`Button`的可访问性特性，支持VoiceOver等辅助功能。
- **材质背景**: 使用`.background(.regularMaterial, in: Circle())`创建半透明的毛玻璃效果背景，增强视觉层次。
- **边框叠加**: 通过`.overlay`添加细边框，提升按钮的视觉边界。

**关键属性**:
- `public let systemName: String`: 图标系统名称，用于显示SF Symbols图标。
- `public let accent: Color?`: 强调色，用于自定义图标颜色。
- `public let action: () -> Void`: 点击时执行的回调函数。

**实现细节**:
- `RoundIconView`负责渲染图标和背景，`RoundIconButton`则将其包装在`Button`中处理交互。
- 使用`#available(iOS 17.0, *)`检查iOS版本，为新版本使用`foregroundStyle`，旧版本使用`foregroundColor`。

## 可选标签芯片 (SelectableChip)

可选标签芯片是一种用于选择和显示标签的交互式组件，支持多种变体以适应不同场景。

**核心特性**:
- **选中状态管理**: 通过`isSelected: Bool`参数控制芯片的选中状态，选中时背景色变为强调色，文字变为白色。
- **多种变体**: 提供了`CompactActivityChip`和`CompanionChip`两种变体，分别用于活动和同伴类型的显示。
- **视觉反馈**: 选中状态通过背景色和文字颜色的变化提供清晰的视觉反馈。

**关键属性**:
- `let text: String`: 显示的文本内容。
- `let icon: String?`: 可选的图标系统名称。
- `let isSelected: Bool`: 当前是否被选中。
- `let accent: Color`: 选中时的背景强调色。
- `let action: () -> Void`: 点击时执行的回调函数。

**使用场景**:
- 在`MindStateFlowScreen`中，`TagChip`用于选择情绪标签。
- 在活动追踪中，`CompactActivityChip`用于选择活动类型。

## 粗滑块 (ThickSlider)

粗滑块是一种专为情绪强度调节设计的自定义滑块组件，提供直观的视觉反馈。

**核心特性**:
- **情绪强度调节**: 主要用于`MindState`模块，让用户选择当前的情绪强度值（0-100）。
- **视觉反馈**: 滑块的填充部分颜色会根据情绪强度动态变化，提供直观的视觉反馈。
- **自定义拖动手势**: 使用`DragGesture`实现自定义的拖动逻辑，精确控制滑块值。

**关键属性**:
- `@Binding public var value: Double`: 双向绑定的滑块值。
- `public let range: ClosedRange<Double>`: 滑块值的范围。
- `public let step: Double`: 滑块值的步长。
- `public let leftText: String`: 左侧标签文本。
- `public let rightText: String`: 右侧标签文本。
- `public let accent: Color`: 滑块的强调色。

**实现机制**:
1. 使用`GeometryReader`获取滑块容器的宽度。
2. 计算滑块手柄的位置，基于当前值在范围内的比例。
3. 使用`ZStack`堆叠两个`Capsule`，一个作为背景，一个作为填充。
4. 通过`DragGesture`监听拖动事件，计算新的值并更新`@Binding`。

**使用示例**:
```swift
// 文件路径: Features/MindState/MindStateFlowScreen.swift
ThickSlider(value: $vm.valenceValue, range: 0...100, step: 1, leftText: Localization.tr(MindValence.veryUnpleasant.titleKey), rightText: Localization.tr(MindValence.veryPleasant.titleKey), accent: valenceAccent(vm.valenceSegment))
```

## 标签输入芯片 (TagInputChip)

标签输入芯片是一种用于显示和管理用户创建标签的组件，具有独特的视觉标识。

**核心特性**:
- **删除交互**: 虽然代码中未直接实现删除功能，但通过`action`回调，父组件可以处理点击事件，实现删除逻辑。
- **用户创建标识**: 使用填充的星形图标`"star.fill"`作为用户创建标签的视觉标识，区别于系统预设标签。
- **选中状态**: 支持选中状态，选中时背景色变为强调色。

**关键属性**:
- `let tag: ActivityTag`: 绑定的标签模型。
- `let isSelected: Bool`: 当前是否被选中。
- `let action: () -> Void`: 点击时执行的回调函数。

**相关组件**:
- `SimpleTagChip`: 简化版本的标签芯片，支持是否显示星形图标。
- `AddTagButton`: 用于添加新标签的按钮组件。

**使用场景**:
- 在`ContextDetailSheet`中，用于显示和选择活动标签。
- 用户可以通过点击`AddTagButton`创建新标签，然后通过`TagInputChip`进行管理。

## 组件复用模式

这些原子组件在观己应用的不同功能模块中被广泛复用，体现了高内聚、低耦合的设计原则。

### 输入系统
- **CapsuleTextEditor**: 作为主要的文本输入组件，集成在`InputDock`和`ExpandedInputView`中，提供统一的输入体验。
- **RoundIconButton**: 用于输入栏中的各种操作按钮，如添加照片、录音等。

### 追踪面板
- **ThickSlider**: 在`MindStateFlowScreen`中用于调节情绪强度，在`DailyTrackerFlowScreen`中用于调节身体能量和心情天气。
- **SelectableChip**: 用于选择活动类型和同伴类型，提供直观的多选界面。

### 数据展示
- **TagInputChip**: 在`ContextDetailSheet`等详情页面中，用于展示和管理标签，支持用户自定义分类。
