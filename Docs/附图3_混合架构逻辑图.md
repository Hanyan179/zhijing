## 设计哲学

### 1. 概率性 vs 确定性
- **AI (概率性)**：适合处理语义理解、情感分析、内容生成等"模糊"任务
- **代码 (确定性)**：适合处理数据校验、流程控制、状态管理等"精确"任务

### 2. 发散性 vs 收敛性
- **AI (发散性)**：从一段文字中提取多个可能的标签、情绪、意图
- **代码 (收敛性)**：将 AI 的发散输出收敛为符合 Schema 的结构化数据

### 3. 容错边界
- AI 输出不可控 → 代码层做严格校验
- 网络请求可能失败 → 代码层做重试和降级
- 数据格式可能变化 → 代码层做版本迁移

### 4. 实际收益
- **开发效率**：不再纠结于复杂的正则表达式和规则引擎
- **用户体验**：AI 理解自然语言，用户无需学习特定格式
- **系统稳定**：代码兜底确保数据一致性和系统可靠性
